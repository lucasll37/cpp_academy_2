# CONTRIBUTING

Este repositório segue práticas institucionais de desenvolvimento.

## Regras Essenciais

- A branch **main** é protegida.  
  Não são permitidos commits diretos.

- O desenvolvimento segue **GitHub Flow**.  
  Toda alteração deve ocorrer em branch própria e ser integrada via *Merge Request*.

- Commits devem seguir **Conventional Commits**  
  (`feat:`, `fix:`, `docs:`, etc.).

- *Merge Requests* devem:
  - passar pelos pipelines de validação;
  - ser revisados antes do merge;
  - descrever claramente o que foi feito e como validar.

## Organização

- O projeto utiliza **SCRUM**.
- Um **Kanban técnico** concentra as tarefas de desenvolvimento e é a principal referência para novas contribuições.

## Documentação

- Documentação técnica prioritariamente em **LaTeX**.
- Markdown/RST podem ser utilizados quando apropriado.

Ao contribuir, você concorda com estas diretrizes.
